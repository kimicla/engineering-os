# Principles

1. Current Thinking is the single source of truth.

2. Daily notes are for execution, not long-term documentation.

3. Weekly Snapshot is the review layer that updates Current Thinking.

4. AI proposes. I decide.

5. Optimize for decision quality, not note quality.

6. Prefer removing constraints over completing tasks.

7. Prefer architectural consistency over implementation perfection.

8. The system should reduce cognitive load.

9. Any process that cannot be sustained should be rejected.

10. Simplicity beats completeness.

## Working rule

A change is only valid when it solves an observed problem.

A good idea without a real usage problem goes to the backlog, not directly into the system.
